## 正式角色

- 当前文件名: `00_技能体系导航.md`
- 本文件只负责项目内 skill/template/规程分层索引
- 本文件不是根目录总入口
- 本文件没有独立放行权
- 本文件必须与根目录 `crc-gland-coding-guard` 保持同一执行顺序

这份文件只回答 3 个问题:

1. 项目内哪些文件应该保留
2. 每个文件只负责什么
3. 当前阶段应该读哪些文件

## 唯一原则

- 根目录唯一总入口: `d:\12_Medical_Image_Segmentation\Paper\.trae\skills\crc-gland-coding-guard\SKILL.md`
- 根目录 `standard-md-rewrite` 只负责 `结直肠腺体分割_plan_优化版/01_实验执行` 的 plan 正文重写协议,不参与代码工作流路由,也不替代任何 gate report
- 项目内 `.trae/skills/` 只允许做被动模板层和规程层
- 项目内 B 类真实工具与运行留痕固定集中到 `crc_gland_segmentation_project/b_class_auxiliary/`; 不再与 `tools/` 主区或 `reports/stage_reports/` 混放
- 根目录 `tools/` 采用阶段化主区: A 类正式工具进入 `tools/stageXX_[阶段名]/`, C 类待裁决工具进入 `tools/c_pending_review/`
- 固定执行链只有一条: `治理 -> 研究 -> 阶段锁定 -> Pre-check -> 编码 -> 运行证据 -> 代码质量 -> 总放行 -> 说明文档 -> 交付前清理`
- `workflow_gate_report.md` 只负责“本轮正式任务是否整体通过”,不替代研究、阶段锁定或 Pre-check
- 一个文件属不属于项目,以 `制度完成定义.md`「文件三分归属」为唯一权威: A(项目正式对象,交付+写说明文) / B(AI 辅助与流程文件,不交付+不写说明文) / C(存疑待清理); 禁止出现既算项目又算辅助的混合件

## 状态口径

- `Stage Gate`: `allow / blocked`
- `Diagnostics / Post-QC`: `pass / partial / fail / not_applicable`
- `implementation_tracking`: `not_started / blocked / partial / pass`

## 执行透明度

项目内导航层必须强制一件事:

- 每次开始任务、恢复任务或收到“继续”时,先显式展示本轮已遍历的 skill / 导航 / 规程 / 当前阶段正式文件清单

最小展示口径:

- `治理`: `crc-gland-skill-governance`、`crc-gland-coding-guard`、本文件、`制度完成定义.md`、`项目阶段推进指令使用手册.md`
- `研究`: 根目录总入口、研究 skill、当前阶段计划文件、正式参考资料入口、最近 `research_alignment_gate_report.md`
- `阶段锁定 / Pre-check / 编码 / 总放行 / 说明文档`: 根目录总入口、对应阶段 skill 或规程、当前阶段正式协议/记录、最近一次 gate report

进入 `说明文档` 阶段时,展示口径再加两条硬要求:

- 必须显式区分“当前编号阶段”和“当前 skill 工作流阶段”,例如 `当前编号阶段=01_数据协议`, `当前工作流阶段=说明文档`
- 必须显式列出“本轮交付范围内正式对象清单”,不能只列 gate / guard / skill / 导航文件

如果这两条没有做到,不允许宣称“当前已经开始或完成项目文件说明”。

如果清单不全,先补读,不要先给阶段结论。

## 强制规则

- 研究阶段必须回链 `模板R_研究定标记录.md` 并产出研究定标记录
- 阶段锁定必须回链 `模板0_阶段实现卡.md` 并产出 `stage_definition_gate_report.md`
- Pre-check 必须回链 `b_class_auxiliary/tools/check_precheck_docs.py`
- 每次阶段判断前必须先展示本轮已遍历文件清单
- 涉及正式代码改动时,必须先回链 `模板I_实现依据记录.md` 并生成或更新 `实现依据记录.md`
- 涉及正式代码改动时,必须同时在 `reports/stage_reports/implementation_tracking/[当前阶段名]/实现依据记录.md` 保留当前阶段独立归档; 根路径 `b_class_auxiliary/runtime_checks/实现依据记录.md` 只能作为当前阶段副本
- 新阶段第一次进入编码前,应优先使用 `b_class_auxiliary/tools/init_stage_archive.py` 初始化阶段归档目录与 `实现依据记录.md` 初稿
- 涉及正式代码改动时,必须回链 `b_class_auxiliary/tools/run_minimal_runtime_check.py` 和 `b_class_auxiliary/tools/check_code_quality_gate.py`
- 准备宣称本轮正式任务整体通过时,必须回链 `b_class_auxiliary/tools/enforce_workflow_gate.py` 并生成 `workflow_gate_report.md`
- 学习型说明文写作前必须回链 `b_class_auxiliary/tools/check_learning_docs.py`
- 正式模板或规程修改后必须回链 `b_class_auxiliary/tools/check_formal_stage_docs.py`
- 当前仍在治理,或当前编号阶段的正式产物与 gate 未收口时,不得把“下一编号阶段”写成当前结论
- 进入 `说明文档` 阶段时,不得用 `README.md` / `implementation_status.md` 代替新增正式 Python / YAML / CSV / 资产的逐文件说明
- 进入 `说明文档` 阶段时,不得把 `.trae/`、guard、gate、diagnostics 等内部留痕写成“项目相关文件说明”
