## 为什么当前是这 `50` 个

当前这批对象被纳入，是因为它们正好构成 `02_UNet流程验证` 的最小可审计闭环:

1. 训练入口、验证入口、可视化入口都在其中。
2. 配置冻结来源都在其中。
3. data-model-loss-engine-eval 主链都在其中。
4. 主 run 的 full run 正式资产都在其中。
5. 规范 smoke run 的对照资产也在其中。

尤其要强调主 run 的资产口径已经变化:

1. `run_meta.yaml` 现在记录的是 `smoke_check=false`、`best_epoch=48`、`epoch_count=68`、`stage_pass=true`。
2. `train_log.csv` 和 `val_metrics.csv` 现在都是 `1..68` 的长程正式记录。
3. `run_summary.md` 现在已经是 full run 的正式摘要页。
4. `testA_metrics.csv`、`testB_metrics.csv`、`metric_crosscheck_note.md`、`error_cases.md` 已把测试与可视化链补齐。

因此，当前主目录不再属于“最小 smoke run 资产”。

1. 把新对象纳入清单裁决。
2. 补源码来源 docstring。
3. 补对象级说明文。
4. 同步更新 `README.md` 和 `implementation_status.md`。

所以当前能成立的诚实结论是:

1. 当前锁定范围内的 `50` 个 A 类对象已经纳入正式说明文覆盖。
2. 当前主 run 的说明文已经和 full run 现实对齐。
