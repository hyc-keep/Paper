## 2. 按 PDF 建卡

### 2.1 `GlaS_Challenge_Contest_2017.pdf`

用途定位：

- 当前项目最核心的数据集与评价协议来源之一。
- 主要用于写 `GlaS` 数据来源、官方划分、官方评价指标、边界和对象级评价的临床意义。

可直接提取的关键信息：

1. `GlaS` 挑战的目标是结直肠组织病理图像中的腺体分割，任务背景直接连接到结直肠癌分级和腺体形态学分析。
   - 来源：`GlaS_Challenge_Contest_2017.pdf`, p.2

2. 数据集由训练集、测试 `Part A`、测试 `Part B` 组成，并按良性/恶性分层；表格给出了各部分图像数量与尺寸。
   - 来源：`GlaS_Challenge_Contest_2017.pdf`, p.4

3. 图像来自 `H&E` 切片，在 `20×` 放大倍率下采集；数据切分时按视野而不是按患者严格分层，因此同一张 slide 的不同视野可能落在不同子集中。
   - 来源：`GlaS_Challenge_Contest_2017.pdf`, p.6

4. 官方检测指标采用 `F1 score`，并把与对应真值目标重叠至少 `50%` 的预测实例记为真阳性。
   - 来源：`GlaS_Challenge_Contest_2017.pdf`, p.8

5. 官方分割指标不是简单像素 Dice，而是 `object-level Dice`，强调个体腺体级别的一致性。
   - 来源：`GlaS_Challenge_Contest_2017.pdf`, p.9

6. 官方边界相关指标采用 `object-level Hausdorff distance`，明确用于衡量边界形状相似性。
   - 来源：`GlaS_Challenge_Contest_2017.pdf`, p.10

7. 文中明确说明：`Object Hausdorff` 对形状相似性更有临床相关性，`Object Dice` 更反映单个腺体体积接近程度，`F1` 更反映能否找到尽可能多的腺体实例。
   - 来源：`GlaS_Challenge_Contest_2017.pdf`, p.22

对当前实验最有用的落地点：

- 后面实验主表应至少包含 `F1 / Object Dice / Object Hausdorff(或 HD 变体)` 这一类对象级和边界级指标，而不是只报像素 Dice。
- 写数据集时必须注明 `20×`、按视野分层而非严格按患者分层。

### 2.2 `U-Net_2015.pdf`

用途定位：

- 当前项目最基础的分割 baseline 来源。
- 用于写 `U-Net` baseline 合理性、编码器-解码器结构、数据增强和边界加权思想。

可直接提取的关键信息：

1. `U-Net` 的核心思想是收缩路径获取上下文、对称扩张路径实现精确定位，且在少量标注样本下依赖强数据增强完成训练。
   - 来源：`U-Net_2015.pdf`, p.1

2. 标准 `U-Net` 是典型的 encoder-decoder 结构，采用跳连连接编码器和解码器特征，原文结构图就是后面所有 `U-Net` 系方法的最经典出处。
   - 来源：`U-Net_2015.pdf`, p.2, p.4

3. 原文强调大图分块时使用 overlap-tile 策略，可无缝处理任意大图像。
   - 来源：`U-Net_2015.pdf`, p.3

4. 训练时使用对边界和细缝区域加权的像素损失图，以迫使网络学习相邻目标之间的分离边界。
   - 来源：`U-Net_2015.pdf`, p.5

5. 对小样本医学分割，`elastic deformation` 被原文明确作为关键增强方式。
   - 来源：`U-Net_2015.pdf`, p.6

6. 训练细节里，作者明确更偏向“大输入 tile 而不是大 batch”，因此实际采用 `batch size = 1` 并配合 `momentum = 0.99` 的 `SGD`；这说明在医学分割里，上下文覆盖有时比常规大 batch 训练更重要。
   - 来源：`U-Net_2015.pdf`, p.4

对当前实验最有用的落地点：

- `U-Net` 可以作为第一个“流程跑通 baseline”。
- 你后面如果在损失或后处理中特别强调分离黏连腺体，`U-Net` p.5 的边界加权思想是很好的早期出处。

### 2.3 `Deep_Residual_Learning_for_Image_Recognition_2016.pdf`

用途定位：

- `ResNet34-U-Net` 主 baseline 的 encoder/backbone 出处。
- 不是腺体论文，但它是“为什么能把 U-Net 编码器换成 ResNet”的最标准来源。

可直接提取的关键信息：

1. `ResNet` 的关键思想是把学习目标改写成残差映射 `F(x)=H(x)-x`，并通过 shortcut identity connection 缓解深层网络优化困难。
   - 来源：`Deep_Residual_Learning_for_Image_Recognition_2016.pdf`, p.2, p.3

2. 原文给出了 `34-layer residual network` 的标准结构，这正是后面 `ResNet34-U-Net` 中 `ResNet34` 的直接出处。
   - 来源：`Deep_Residual_Learning_for_Image_Recognition_2016.pdf`, p.4, p.5

3. 残差结构相比 plain network 更容易优化，`34-layer ResNet` 相比 plain-34 显著降低验证误差。
   - 来源：`Deep_Residual_Learning_for_Image_Recognition_2016.pdf`, p.5, p.6

4. 原文还强调 `identity shortcut` 几乎不增加额外参数和计算量，因此 plain net 与 residual net 可以在相近复杂度下做公平比较；这也是后来把 `ResNet` 当作 `U-Net` 编码器 backbone 的工程优势之一。
   - 来源：`Deep_Residual_Learning_for_Image_Recognition_2016.pdf`, p.3, p.4

对当前实验最有用的落地点：

- 如果你把主 baseline 定成 `ResNet34-U-Net`，方法部分的 backbone 说明就引用这篇。
- 这篇的作用不是解释分割任务，而是解释“为什么用残差编码器”。

### 2.4 `UNet++_2018.pdf`

用途定位：

- 经典外部对比方法之一。
- 用于解释“比 U-Net 更强的 skip connection 设计”和深监督改进。

可直接提取的关键信息：

1. `UNet++` 通过重设计 skip pathway 和深监督，目标是缩小 encoder 与 decoder 特征之间的语义差距。
   - 来源：`UNet++_2018.pdf`, p.1, p.2

2. 原文结构的核心区别在于嵌套式、密集式 skip connection，而不是简单平直跳连。
   - 来源：`UNet++_2018.pdf`, p.3, p.4

3. 论文明确使用 `binary cross-entropy + Dice coefficient` 作为损失组合。
   - 来源：`UNet++_2018.pdf`, p.5

4. 实验中 `UNet++ with deep supervision` 相比 `U-Net` 和 `wide U-Net` 取得更高 `IoU`。
   - 来源：`UNet++_2018.pdf`, p.6, p.7, p.8

5. `deep supervision` 不只是提精度，还允许 `accurate mode / fast mode` 两种推理方式；文中 `UNet++ L3` 剪枝后平均推理时间下降约 `32.2%`，而 `IoU` 只下降 `0.6`，说明它可以在速度和精度之间做结构化权衡。
   - 来源：`UNet++_2018.pdf`, p.4, p.7

对当前实验最有用的落地点：

- 后面做外部对比时，`UNet++` 是非常合理的“U-Net 系增强版”代表。
- 如果你后面写 related work，可以把它归为“通过 skip pathway redesign 改善信息融合”的经典工作。

### 2.5 `DeepLabV3+_Encoder-Decoder_with_Atrous_Separable_Convolution_2018.pdf`

用途定位：

- 强外部对比方法之一。
- 用于支持“大感受野 + encoder-decoder + 边界细化”的外部通用基线。

可直接提取的关键信息：

1. `DeepLabv3+` 的主张是把 `ASPP` 的多尺度上下文与 encoder-decoder 的边界恢复能力结合起来。
   - 来源：`DeepLabV3+_Encoder-Decoder_with_Atrous_Separable_Convolution_2018.pdf`, p.1, p.2

2. 原文明确说 decoder 的作用是“reﬁne the segmentation results especially along object boundaries”。
   - 来源：`DeepLabV3+_Encoder-Decoder_with_Atrous_Separable_Convolution_2018.pdf`, p.1, p.4

3. 论文指出 encoder 通常使用 `output stride = 16`，若用 `8` 会更精细但计算代价更高，这是很典型的精度-复杂度权衡描述。
   - 来源：`DeepLabV3+_Encoder-Decoder_with_Atrous_Separable_Convolution_2018.pdf`, p.6, p.9, p.10

4. 低层特征在 decoder 里先用 `1×1 conv` 降通道，再与高层语义特征拼接，后接两个 `3×3 conv` 细化边界。
   - 来源：`DeepLabV3+_Encoder-Decoder_with_Atrous_Separable_Convolution_2018.pdf`, p.6, p.8, p.9

5. 论文在 `PASCAL VOC 2012` 和 `Cityscapes` 上分别达到 `89.0%` 与 `82.1%`，并明确强调结果是 `without any post-processing`，说明其边界恢复能力主要来自网络本身而不是额外 `CRF` 一类后处理。
   - 来源：`DeepLabV3+_Encoder-Decoder_with_Atrous_Separable_Convolution_2018.pdf`, p.1, p.3

对当前实验最有用的落地点：

- `DeepLabV3+` 很适合作为你主结果表里的强通用外部对比。
- 如果你后面想论证“大感受野和边界恢复都重要”，它是很好的对比参照。

### 2.6 `Attention_U-Net_2018.pdf`

用途定位：

- 外部对比方法之一。
- 用于说明 attention gate 可以在 skip connection 上筛掉无关响应，增强对目标区域的聚焦。

可直接提取的关键信息：

1. `Attention U-Net` 的核心是 attention gate，能在不显著增加计算量的情况下突出对目标结构有用的响应。
   - 来源：`Attention_U-Net_2018.pdf`, p.1

2. 原文明确指出 attention gate 可以替代某些额外的显式定位模块。
   - 来源：`Attention_U-Net_2018.pdf`, p.1, p.2

3. 注意力模块是加在 skip connection 上，对传递到解码器的特征进行筛选。
   - 来源：`Attention_U-Net_2018.pdf`, p.3, p.5

4. 原文实验显示，相比标准 `U-Net`，注意力机制提高了分割性能，且参数增量不大。
   - 来源：`Attention_U-Net_2018.pdf`, p.7

5. 论文把 `AG` 的作用概括为逐步抑制无关背景区域的特征响应，从而提升 foreground sensitivity；这意味着它试图用网络内部注意力来替代级联 `ROI/localization` 框架。
   - 来源：`Attention_U-Net_2018.pdf`, p.2, p.4

对当前实验最有用的落地点：

- 适合作为“轻量注意力增强型 U-Net”对比。
- 如果你最终不做显式 attention 模块，这篇依然可以放在 external comparison 和 related work 里。

### 2.7 `DCAN_Deep_Contour-Aware_Networks_for_Accurate_Gland_Segmentation_2016.pdf`

用途定位：

- 腺体分割直接相关经典论文。
- 是你做 `Boundary Head` 最关键的早期动机来源之一。

可直接提取的关键信息：

1. `DCAN` 采用 object + contour 的统一多任务框架，既预测腺体区域也预测腺体轮廓，用于分离相互黏连的腺体。
   - 来源：`DCAN_Deep_Contour-Aware_Networks_for_Accurate_Gland_Segmentation_2016.pdf`, p.1

2. 文中指出恶性腺体形态高度变异，单纯依赖先验结构容易失败，因此需要深层上下文和轮廓信息结合。
   - 来源：`DCAN_Deep_Contour-Aware_Networks_for_Accurate_Gland_Segmentation_2016.pdf`, p.2, p.3

3. 论文采用多层上下文特征和辅助监督来增强训练。
   - 来源：`DCAN_Deep_Contour-Aware_Networks_for_Accurate_Gland_Segmentation_2016.pdf`, p.3, p.4

4. 训练输入为随机裁剪的 `480×480` patch，使用平移、旋转、弹性形变等增强。
   - 来源：`DCAN_Deep_Contour-Aware_Networks_for_Accurate_Gland_Segmentation_2016.pdf`, p.5

5. 论文使用 object-level Dice 等对象级指标，并在 GlaS 挑战中名列前茅。
   - 来源：`DCAN_Deep_Contour-Aware_Networks_for_Accurate_Gland_Segmentation_2016.pdf`, p.7, p.8

6. 最终实例掩膜不是直接读取网络输出，而是把 gland/object 概率与 contour 概率按阈值 `0.5` 融合，再做 `disk filter(radius=3)`、填孔和去小区域；说明 contour 分支后的轻量后处理依然会影响最终分离效果。
   - 来源：`DCAN_Deep_Contour-Aware_Networks_for_Accurate_Gland_Segmentation_2016.pdf`, p.4, p.5

7. 为缓解小样本问题，作者使用在 `PASCAL VOC 2012` 上训练的 `DeepLab` 预训练权重初始化下采样路径，并指出这能显著加快收敛。
   - 来源：`DCAN_Deep_Contour-Aware_Networks_for_Accurate_Gland_Segmentation_2016.pdf`, p.5

对当前实验最有用的落地点：

- 这是你后面解释“为什么要加 Boundary Head，而不是只做普通二分类分割”的强引用。
- 如果你做边界分支消融，这篇应放在方法动机和 related work 里。

### 2.8 `MILD-Net_Minimal_Information_Loss_Dilated_Network_2018.pdf`

用途定位：

- 当前项目最关键的腺体分割参考文献之一。
- 同时覆盖 `GlaS`、`CRAG`、边界、感受野、实例分割与不确定性。

可直接提取的关键信息：

1. 论文同时展示 `GlaS` 与 `CRAG` 示例图，明确把二者作为结肠腺体实例分割任务的重要数据来源。
   - 来源：`MILD-Net_Minimal_Information_Loss_Dilated_Network_2018.pdf`, p.2

2. 文中指出腺体分割难点包括边界精细描绘、腺体大小形态变化大，以及腺体与非腺体结构相似。
   - 来源：`MILD-Net_Minimal_Information_Loss_Dilated_Network_2018.pdf`, p.3

3. `MILD-Net` 的核心结构是通过 `MIL unit` 减少 max-pooling 带来的信息损失，并结合 dilated residual unit 与 ASPP 保持分辨率和多尺度感受野。
   - 来源：`MILD-Net_Minimal_Information_Loss_Dilated_Network_2018.pdf`, p.4, p.5

4. 损失函数由 gland、contour 和 auxiliary 分支共同组成，总损失里包含主输出与辅助输出的交叉熵项。
   - 来源：`MILD-Net_Minimal_Information_Loss_Dilated_Network_2018.pdf`, p.6

5. 论文在测试阶段引入随机变换采样 `RTS` 来估计不确定性，并指出高不确定性区域通常就是难分类区域。
   - 来源：`MILD-Net_Minimal_Information_Loss_Dilated_Network_2018.pdf`, p.7

6. 文中明确写出实验数据协议：
   - `GlaS`: 85 train / 80 test
   - `CRAG`: 173 train / 40 test
   - 从训练集中再留出 `20%` 作为训练过程中的评估集
   - patch 提取为 `500×500`，随机裁成 `464×464`
   - 增强包括弹性形变、翻转、旋转、Gaussian blur、median blur、颜色扰动
   - 来源：`MILD-Net_Minimal_Information_Loss_Dilated_Network_2018.pdf`, p.9

7. 文中直接给出 `GlaS` 与 `CRAG` 的对比表，使用 `F1 Score / Object Dice / Object Hausdorff` 作为主要评价。
   - 来源：`MILD-Net_Minimal_Information_Loss_Dilated_Network_2018.pdf`, p.12

8. 论文指出：更大的感受野、边界建模和不确定性处理都有助于腺体形态特征的可靠提取。
   - 来源：`MILD-Net_Minimal_Information_Loss_Dilated_Network_2018.pdf`, p.17

9. 论文不仅把 `RTS` 用于生成 uncertainty map，还明确提出可据此忽略高不确定区域、并为 pathologist annotation 排序；扩展版 `MILD-Net+` 还可同步分割 gland 和 lumen。
   - 来源：`MILD-Net_Minimal_Information_Loss_Dilated_Network_2018.pdf`, p.1, p.3

对当前实验最有用的落地点：

- `CRAG` 的实验协议、数据来源和第二 benchmark 的依据，这篇非常关键。
- 你后面锁 patch size、增强和评价指标时，这篇几乎可以直接作为重要参考模板。

### 2.9 `Gated-SCNN_Gated_Shape_CNNs_for_Semantic_Segmentation_2019.pdf`

用途定位：

- `Boundary Head` 的强动机来源。
- 用于说明“边界/形状信息应作为单独分支建模”这一路线不是拍脑袋想到的。

可直接提取的关键信息：

1. `Gated-SCNN` 明确把 shape information 作为与常规语义流并行的独立分支处理。
   - 来源：`Gated-SCNN_Gated_Shape_CNNs_for_Semantic_Segmentation_2019.pdf`, p.1

2. 论文强调 boundary supervision 和 dual-task regularizer，用于促进分割结果与真实边界对齐。
   - 来源：`Gated-SCNN_Gated_Shape_CNNs_for_Semantic_Segmentation_2019.pdf`, p.2

3. 整体结构由 regular stream、shape stream 和 fusion module 构成，shape stream 重点处理边界信息。
   - 来源：`Gated-SCNN_Gated_Shape_CNNs_for_Semantic_Segmentation_2019.pdf`, p.3

4. Gated Convolution Layer 用高层语义流去门控低层 shape stream 的响应，抑制无关噪声，只保留和边界有关的信息。
   - 来源：`Gated-SCNN_Gated_Shape_CNNs_for_Semantic_Segmentation_2019.pdf`, p.4

5. 加入 GCL 和 dual-task loss 后，边界质量 `F-score` 和整体 `mIoU` 都得到提升。
   - 来源：`Gated-SCNN_Gated_Shape_CNNs_for_Semantic_Segmentation_2019.pdf`, p.6

6. 作者还强调：shape stream 采用的是一个工作在全分辨率上的浅层分支，并由 regular stream 的高层语义来“去噪”；这说明显式边界流不一定要做得很重，关键是让它只处理真正相关的 boundary 信息。
   - 来源：`Gated-SCNN_Gated_Shape_CNNs_for_Semantic_Segmentation_2019.pdf`, p.1, p.2

7. 结果分析指出，该结构对 `thinner and smaller objects` 提升尤其明显；这点对腺体任务里窄边界、细缝和小腺体分离很有借鉴意义。
   - 来源：`Gated-SCNN_Gated_Shape_CNNs_for_Semantic_Segmentation_2019.pdf`, p.1, p.2

对当前实验最有用的落地点：

- 这篇能直接支撑你把 `Boundary Head` 设计成显式分支，而不是只在 loss 上轻描淡写处理边界。

### 2.10 `Distance_Map_Loss_Penalty_Term_for_Semantic_Segmentation_2019.pdf`

用途定位：

- `Distance-aware Loss` 的直接动机来源。
- 用于证明边界附近错误需要被额外惩罚。

可直接提取的关键信息：

1. 论文开宗明义指出：语义分割在目标边界处性能较差，而医学分割特别需要准确保持组织表面和形状。
   - 来源：`Distance_Map_Loss_Penalty_Term_for_Semantic_Segmentation_2019.pdf`, p.1

2. 论文使用由真值 mask 生成的 distance map 作为惩罚项，使网络更关注难分割的边界区域。
   - 来源：`Distance_Map_Loss_Penalty_Term_for_Semantic_Segmentation_2019.pdf`, p.1, p.2

3. 具体损失形式是在多类交叉熵前乘以 `(1 + Φ)`，其中 `Φ` 为 distance map。
   - 来源：`Distance_Map_Loss_Penalty_Term_for_Semantic_Segmentation_2019.pdf`, p.3

4. 论文指出该项在边界 Dice 上有明显提升，同时全局 Dice 不下降。
   - 来源：`Distance_Map_Loss_Penalty_Term_for_Semantic_Segmentation_2019.pdf`, p.4, p.5

5. 该文不是只和 `Dice loss` 比较，而是同时对照了 `focal loss` 和惩罚过度自信预测的正则项，最终边界 `Dice` 仍最好，说明 distance-aware penalty 的收益主要体现在 shape preservation 而不是单纯替代常规损失。
   - 来源：`Distance_Map_Loss_Penalty_Term_for_Semantic_Segmentation_2019.pdf`, p.3, p.4

对当前实验最有用的落地点：

- 你如果给主模型加 `Distance-aware Loss`，这篇就是最直接的方法动机引用。
- 如果后面实验发现主指标收益不稳定，也可以把它当“可选增强项”，不是必须硬塞进主模型。

对当前实验最有用的落地点：

### 2.14 `nnU-Net_a_self-configuring_method_for_deep_learning-based_biomedical_image_segmentation_2021.pdf`

用途定位：

- 强通用医学分割 baseline 与实验规范参考。
- 这篇的价值不只是“多一个模型”，更重要的是告诉你：很多性能差异来自 pipeline 配置，而不只是网络花样。

可直接提取的关键信息：

1. `nnU-Net` 明确提出一组固定设计原则，并直接指出：在 pipeline 配置合理时，`U-Net like architectures` 已经能够达到 state-of-the-art，未必需要复杂结构变体。
   - 来源：`nnU-Net_a_self-configuring_method_for_deep_learning-based_biomedical_image_segmentation_2021.pdf`, p.3

2. 论文自动生成三种配置：`2D U-Net`、`3D full resolution U-Net`、`3D U-Net cascade`，并指出 `3D full resolution U-Net` 整体上表现最好，但具体任务仍需经交叉验证决定。
   - 来源：`nnU-Net_a_self-configuring_method_for_deep_learning-based_biomedical_image_segmentation_2021.pdf`, p.3, p.5

3. 训练损失上，文中明确说单独 `Dice loss` 不够稳定，因此经验上采用 `Dice + cross-entropy` 简单平均来提升训练稳定性和分割精度。
   - 来源：`nnU-Net_a_self-configuring_method_for_deep_learning-based_biomedical_image_segmentation_2021.pdf`, p.4

4. 推理阶段采用与训练相同 patch size 的 `patch-based inference`，相邻 patch 以 `patch_size / 2` 的步长滑动，并用 `Gaussian importance weighting` 缓解拼接边界伪影。
   - 来源：`nnU-Net_a_self-configuring_method_for_deep_learning-based_biomedical_image_segmentation_2021.pdf`, p.4

5. 文中把 `5-fold cross-validation` 训练出的 5 个模型作为 ensemble 预测测试集，并在验证后再决定最佳配置与是否做最大连通域后处理。
   - 来源：`nnU-Net_a_self-configuring_method_for_deep_learning-based_biomedical_image_segmentation_2021.pdf`, p.4, p.5

6. 关于 patch 设计，原文强调 patch size 应“在显存允许下尽可能大”，以保证网络感受野覆盖足够上下文；batch size 最少设为 `2` 以保持优化稳定。
   - 来源：`nnU-Net_a_self-configuring_method_for_deep_learning-based_biomedical_image_segmentation_2021.pdf`, p.5

7. 训练协议上，`nnU-Net` 不是模糊经验，而是固定 `1000 epochs`、`SGD(lr=0.01, Nesterov momentum=0.99)`、`polyLR` 和在线数据增强；这组默认设置很适合拿来校准“baseline 是否训够、训公平”。
   - 来源：`nnU-Net_a_self-configuring_method_for_deep_learning-based_biomedical_image_segmentation_2021.pdf`, p.3, p.4

对当前实验最有用的落地点：

- 这篇很适合用来提醒你：当前项目先把 `patch / aug / infer / postprocess / validation` 协议锁好，收益往往比盲目加新模块更稳定。
- 即使最后不把 `nnU-Net` 作为主对比，文中的配置哲学也很适合用来校准你的 baseline 公平性。

### 2.15 `TransUNet_Transformers_Make_Strong_Encoders_for_Medical_Image_Segmentation_2021.pdf`

用途定位：

- `Transformer` 医学分割外部对比的重要代表。
- 用于支撑“全局依赖 + 精细定位”这条路线为什么需要 `CNN + Transformer + skip connection` 的组合。

可直接提取的关键信息：

1. 论文明确指出：传统 `U-Net` 受卷积局部性限制，难以显式建模 long-range dependency；而 `Transformer` 天然擅长建模全局上下文。
   - 来源：`TransUNet_Transformers_Make_Strong_Encoders_for_Medical_Image_Segmentation_2021.pdf`, p.1, p.2

2. `TransUNet` 的核心思路是把来自 `CNN feature map` 的 tokenized patches 输入 `Transformer` 编码器，从而提取 global context。
   - 来源：`TransUNet_Transformers_Make_Strong_Encoders_for_Medical_Image_Segmentation_2021.pdf`, p.1, p.3, p.4

3. 原文明确分析：如果只做 naive upsampling，`Transformer` 编码后的低分辨率特征会缺失细粒度定位信息，导致分割结果粗糙。
   - 来源：`TransUNet_Transformers_Make_Strong_Encoders_for_Medical_Image_Segmentation_2021.pdf`, p.2, p.5

4. 为弥补这一点，论文采用 `CNN-Transformer hybrid encoder` 和 `cascaded upsampler`，并把高分辨率 CNN 特征通过 skip connection 送回解码器以实现 precise localization。
   - 来源：`TransUNet_Transformers_Make_Strong_Encoders_for_Medical_Image_Segmentation_2021.pdf`, p.2, p.5

5. 比较实验显示：`CUP` decoder 优于直接上采样，而在 `Transformer` 基础上进一步加入 `CNN` 与 `skip-connections` 后，性能继续明显提升。
   - 来源：`TransUNet_Transformers_Make_Strong_Encoders_for_Medical_Image_Segmentation_2021.pdf`, p.6, p.7

6. 实现层面，作者采用 `224×224` 输入、`patch size = 16`、`ImageNet` 预训练的 `R50-ViT`，并对 3D 体数据按 `slice-by-slice` 的 2D 方式推理后再堆叠评估；这对你当前的 2D 腺体分割设置尤其有参考意义。
   - 来源：`TransUNet_Transformers_Make_Strong_Encoders_for_Medical_Image_Segmentation_2021.pdf`, p.6

对当前实验最有用的落地点：

- 如果你后面要把 `TransUNet` 放进主结果表，它能代表“全局依赖建模”的强外部对比。
- 即使不做 Transformer 主线，这篇也能支持一个很重要的判断：全局建模不能以牺牲边界和定位细节为代价。

### 2.16 `Swin-Unet_Unet-like_Pure_Transformer_for_Medical_Image_Segmentation_2022.pdf`

用途定位：

- `Pure Transformer` 医学分割代表方法之一。
- 用于和 `TransUNet` 形成互补：一个是 `CNN + Transformer hybrid`，一个是 `pure Transformer U-shape`。

可直接提取的关键信息：

1. 论文提出 `Swin-Unet`，明确把它定义为 `Unet-like pure Transformer for medical image segmentation`，核心目标是用 Transformer 学习 local-global semantic interaction。
   - 来源：`Swin-Unet_Unet-like_Pure_Transformer_for_Medical_Image_Segmentation_2022.pdf`, p.1

2. 原文反复强调：`CNN` 在 long-range semantic interaction 上存在局限，而层次化 `Swin Transformer` 可作为编码器提取上下文特征。
   - 来源：`Swin-Unet_Unet-like_Pure_Transformer_for_Medical_Image_Segmentation_2022.pdf`, p.1, p.2

3. `Swin-Unet` 的结构是对称的 `Transformer-based encoder-decoder`，decoder 通过 `patch expanding layer` 进行上采样以恢复空间分辨率。
   - 来源：`Swin-Unet_Unet-like_Pure_Transformer_for_Medical_Image_Segmentation_2022.pdf`, p.1, p.3, p.4, p.7

4. 论文明确写出：`skip connection is also effective for Transformer`，并通过实验把它作为最终保留的关键设计。
   - 来源：`Swin-Unet_Unet-like_Pure_Transformer_for_Medical_Image_Segmentation_2022.pdf`, p.3

5. 消融实验显示：`patch expanding` 比双线性插值和转置卷积带来更高分割精度；同时 skip connection 数量增加时，模型性能也同步提升。
   - 来源：`Swin-Unet_Unet-like_Pure_Transformer_for_Medical_Image_Segmentation_2022.pdf`, p.10

6. 在 `Synapse` 对比中，`Swin-Unet` 相比 `TransUNet` 的 `DSC` 只小幅提升，但 `HD` 从 `31.69` 明显降到 `21.55`；作者据此认为 pure Transformer U-shape 对边界预测和过分割抑制更有优势。
   - 来源：`Swin-Unet_Unet-like_Pure_Transformer_for_Medical_Image_Segmentation_2022.pdf`, p.8, p.9

对当前实验最有用的落地点：

- 如果你需要一个更“纯”的 Transformer 对比，`Swin-Unet` 很合适。
- 这篇也强化了一个结论：无论是 CNN 还是 Transformer，解码恢复与 skip 设计都直接影响边界质量。

### 2.17 `Wavelet_Convolutions_for_Large_Receptive_Fields_2024.pdf`

用途定位：

- 大感受野模块的更近期方法学支撑。
- 用于补强 `LKMA` 的动机，不再只停留在“核更大”，而是说明如何更高效地做超大感受野。

可直接提取的关键信息：

1. 论文开篇指出：借助 `Wavelet Transform`，可以在不过度参数化的前提下实现非常大的感受野；对于 `k × k` receptive field，参数量只随 `k` 对数级增长。
   - 来源：`Wavelet_Convolutions_for_Large_Receptive_Fields_2024.pdf`, p.1, p.2

2. `WTConv` 被设计成可直接替换现有网络中 depth-wise convolution 的 `drop-in replacement`，并能带来 multi-frequency response。
   - 来源：`Wavelet_Convolutions_for_Large_Receptive_Fields_2024.pdf`, p.1, p.3

3. 论文强调，`WTConv` 能获得更大的 `effective receptive field (ERF)`，同时提升对低频和形状信息的响应，表现为更强 `shape bias` 与更好的图像 corruption robustness。
   - 来源：`Wavelet_Convolutions_for_Large_Receptive_Fields_2024.pdf`, p.2, p.3

4. 方法上，作者先在 wavelet domain 中分离不同频带，再对各频带做小核卷积并通过逆小波变换聚合，从而让小核在原图上对应更大的有效感受野。
   - 来源：`Wavelet_Convolutions_for_Large_Receptive_Fields_2024.pdf`, p.5, p.6

5. 文中进一步总结：若固定每层 kernel size，`WT` 的级联分解会使参数量随分解层数线性增长，而感受野按指数级扩大。
   - 来源：`Wavelet_Convolutions_for_Large_Receptive_Fields_2024.pdf`, p.7

6. 作者不仅在分类中验证 `WTConv`，还把它放进 `UperNet` 做语义分割、放进 `Cascade Mask R-CNN` 做检测；说明它更像一个通用 backbone 模块，而不只是某篇分类论文里的专用 trick。
   - 来源：`Wavelet_Convolutions_for_Large_Receptive_Fields_2024.pdf`, p.3

对当前实验最有用的落地点：

- 这篇非常适合用来支撑你后面的大核模块不是“暴力堆大卷积”，而是“更高效地扩大形态感受野”。
- 对腺体任务来说，它还额外提供了“更关注形状而非纹理”的论证点，这和腺体轮廓、腺腔结构很契合。

### 2.18 `SCAU-Net_Spatial-Channel_Attention_U-Net_for_Gland_Segmentation_2020.pdf`

用途定位：

- 腺体任务的 `attention U-Net` 直接参考论文之一。
- 用于支撑空间注意力、通道注意力以及边缘关注在 `GlaS / CRAG` 上的实际有效性。

1. 论文直接在摘要中说明：`SCAU-Net` 是面向 gland segmentation 的 `Spatial-Channel Attention U-Net`，并在 `GlaS` 与 `CRAG` 上优于经典 `U-Net`。

4. 论文明确写到：模型会“pay more attention on the edges of the glands”，因为模糊边界正是最值得关注的分割难点。
   - 来源：`SCAU-Net_Spatial-Channel_Attention_U-Net_for_Gland_Segmentation_2020.pdf`, p.3

5. 这篇评估并没有沿用 `GlaS Challenge` 的对象级三指标，而是按 semantic segmentation 口径使用 `Dice / Jaccard / RVD`；因此它更适合作为 attention 模块有效性的补充证据，不宜与采用 `F1 / object-level Dice / object-level Hausdorff` 的论文直接横向对齐。
   - 来源：`SCAU-Net_Spatial-Channel_Attention_U-Net_for_Gland_Segmentation_2020.pdf`, p.5, p.6

6. 训练协议上，作者把输入统一为 `512×512`，使用旋转、缩放、裁剪等增强，并采用 `0.5 * cross-entropy + 0.5 * Dice loss`、`Adam(lr=1e-4)`、`batch size=4`、`100 epochs`。
   - 来源：`SCAU-Net_Spatial-Channel_Attention_U-Net_for_Gland_Segmentation_2020.pdf`, p.5

7. 可视化结果显示，`SCAU-Net` 相比 `U-Net` 能更好保持 gland object connectivity、抑制背景噪声，并减少多个 gland “sticking” 的情况。
   - 来源：`SCAU-Net_Spatial-Channel_Attention_U-Net_for_Gland_Segmentation_2020.pdf`, p.6

对当前实验最有用的落地点：

- 这篇很适合放进 gland-specific 外部对比或 related work，因为它不是泛医学分割，而是直接针对腺体任务。
- 如果你最后不采用显式 attention 模块，它仍然可以作为“边缘和相关特征筛选很重要”的任务内证据。

### 2.19 `AttentionBoost_Learning_What_to_Attend_for_Gland_Segmentation_2020.pdf`

用途定位：

- 腺体实例分割的多阶段注意力路线参考。
- 主要用于补强对象级评价、实例级后处理和“逐阶段纠错”的思路。

可直接提取的关键信息：

1. 该文把 `AttentionBoost` 定义为一种 error-driven、multi-stage 的 gland instance segmentation 方法，通过 boosting 学习每个 stage 该关注哪些像素。

2. 论文明确沿用 `GlaS Challenge` 的三类核心指标：`object-level F-score`、`object-level Dice`、`object-level Hausdorff distance`，并给出了其定义。
   - 来源：`AttentionBoost_Learning_What_to_Attend_for_Gland_Segmentation_2020.pdf`, p.1

3. 在 gland instance segmentation 步骤中，模型不是简单阈值切图，而是基于平均概率图、seed region、生长与阈值参数去定位实例。

对当前实验最有用的落地点：

### 2.20 `TA-Net_Topology-Aware_Network_for_Gland_Segmentation_2022.pdf`

用途定位：

- 腺体任务里很关键的一篇 `topology-aware` 直接相关工作。
- 尤其适合支撑“仅靠 contour/boundary 可能不够，拓扑结构约束也有价值”这条思路。

可直接提取的关键信息：

1. 摘要直接指出：`TA-Net` 的目标是更好地分离 `densely clustered` 和 `severely deformed` glands，并采用 multitask learning 同时学习 `instance segmentation` 与 `gland topology estimation`。
   - 来源：`TA-Net_Topology-Aware_Network_for_Gland_Segmentation_2022.pdf`, p.1

2. 论文明确批评已有 contour-based gland segmentation 路线：当腺体高度黏连、标注轮廓粗糙、恶性腺体边界扭曲时，单纯 contour strategy 效果有限。
   - 来源：`TA-Net_Topology-Aware_Network_for_Gland_Segmentation_2022.pdf`, p.2

3. 方法核心是引入 gland topology，而不是只做 contour 分割；作者认为 gland 的 `topological skeleton` 在密集黏连区域比 contour 更稳定、更能区分不同腺体。
   - 来源：`TA-Net_Topology-Aware_Network_for_Gland_Segmentation_2022.pdf`, p.2, p.3

4. `TA-Net` 由一个 encoder 和两个 decoder 组成，分别负责 gland instance segmentation 与 topology estimation；其中 topology 分支预测 `Medial Axis (MA) distance map`。
   - 来源：`TA-Net_Topology-Aware_Network_for_Gland_Segmentation_2022.pdf`, p.3, p.4

5. 总损失由 `instance loss + alpha * topology loss` 构成，而 topology loss 又由 `MA distance map loss` 与 `marker loss` 组成；marker loss 主要用来抑制 over-segmentation 和 under-segmentation。
   - 来源：`TA-Net_Topology-Aware_Network_for_Gland_Segmentation_2022.pdf`, p.4, p.5

6. 数据与评估部分直接使用：
   - `CRAG`: `213` 张图像，训练 `173`，测试 `40`
   - `GlaS`: `165` 张图像，训练 `85`，测试 `60 + 20`
   - 指标：`F1 / object-level Dice / object-level Hausdorff`
   - 来源：`TA-Net_Topology-Aware_Network_for_Gland_Segmentation_2022.pdf`, p.5

7. 总结果表显示：`TA-Net` 在 `CRAG` 上取得 `F1 84.2 / Obj-D 89.3 / Obj-H 105.2`，在 `GlaS` 上达到 `F1 90.5 / Obj-D 90.2 / Obj-H 50.8`，说明 topology-aware 路线对黏连腺体更密集的 `CRAG` 尤其有帮助。
   - 来源：`TA-Net_Topology-Aware_Network_for_Gland_Segmentation_2022.pdf`, p.6

8. 距离图消融中，`MA distance` 明显优于 `Euclidean` 和 `Chessboard` 距离，说明保留拓扑结构的距离表征确实更适合这类 gland separation 任务。
   - 来源：`TA-Net_Topology-Aware_Network_for_Gland_Segmentation_2022.pdf`, p.8

对当前实验最有用的落地点：

- 这篇很适合放在 `Boundary Head` 之外的另一条思路旁边，提醒你“分离黏连腺体”不只有边界路线，还有拓扑路线。
- 如果你最后不做拓扑分支，这篇仍然必须在 related work 里占一席，因为它和你的任务难点高度一致。

### 2.21 `Gland_Segmentation_via_Dual_Encoders_and_Boundary-Enhanced_Attention_2024.pdf`

用途定位：

- 当前与你方案最接近的近期腺体分割参考之一。
- 同时覆盖双编码器、边界增强、`GlaS / CRAG`、以及恶性腺体和黏连腺体这两个最关键难点。

可直接提取的关键信息：

1. 论文开篇直接点出腺体分割三大难点：恶性腺体严重形变、相邻腺体黏连、腺体轮廓与大小差异大。
   - 来源：`Gland_Segmentation_via_Dual_Encoders_and_Boundary-Enhanced_Attention_2024.pdf`, p.1

2. 方法上提出 `DEA-Net`，由主干 encoder-decoder 与 `local semantic extraction network` 两个分支组成，其中局部语义引导分支采用预训练 `DeepLabv3+` 提取边缘相关特征。
   - 来源：`Gland_Segmentation_via_Dual_Encoders_and_Boundary-Enhanced_Attention_2024.pdf`, p.1, p.2

3. 论文设计了 `Feature Fusion Module (FFM)` 融合双编码器特征，并在解码端使用 `Boundary-Enhanced Attention (BEA)` 强化腺体边界恢复。
   - 来源：`Gland_Segmentation_via_Dual_Encoders_and_Boundary-Enhanced_Attention_2024.pdf`, p.2, p.3

4. 数据协议部分明确给出：
   - `GlaS`: 165 张图像，训练 `85`，测试 `60 + 20`
   - `CRAG`: 213 张图像，训练 `173`，测试 `40`
   - 来源：`Gland_Segmentation_via_Dual_Encoders_and_Boundary-Enhanced_Attention_2024.pdf`, p.3

5. 训练与评估部分说明采用 challenge 常用的 `F1 / object-level Dice / object-level Hausdorff distance`，并在消融中验证 `LD / FFM / BEA / DFB` 对性能提升都有贡献。
   - 来源：`Gland_Segmentation_via_Dual_Encoders_and_Boundary-Enhanced_Attention_2024.pdf`, p.4

6. 对比实验中，`DEA-Net` 在 `GlaS` 与 `CRAG` 上都取得较强结果，尤其在 `CRAG` 上 `F1` 与 `Dice` 超过多种已有 gland-specific 方法。
   - 来源：`Gland_Segmentation_via_Dual_Encoders_and_Boundary-Enhanced_Attention_2024.pdf`, p.4

7. 训练协议上，作者采用 `1000 epochs`、`Adam(lr=5e-4)`、`batch size=4`，并使用翻转、仿射、随机弹性变换和随机裁剪；同时为每个 gland 构建 `triple mask`，再用 `cross-entropy` 做监督。
   - 来源：`Gland_Segmentation_via_Dual_Encoders_and_Boundary-Enhanced_Attention_2024.pdf`, p.3, p.4

8. 结果解读上也要注意：`DEA-Net` 在 `CRAG` 上虽取得更高 `F1` 和 `Dice`，但 `Hausdorff` (`129.4`) 仍劣于 `TA-Net` (`105.2`)；说明它对实例找全和体积重叠更强，但边界形状相似性未必最优。
   - 来源：`Gland_Segmentation_via_Dual_Encoders_and_Boundary-Enhanced_Attention_2024.pdf`, p.4

对当前实验最有用的落地点：

- 这篇几乎可以直接拿来对照你当前的方案动机：局部语义边缘信息、边界增强、以及针对恶性/黏连腺体的分离。
- 也因此要注意避免和它过于同构，后面写方案时需要明确你的模块差异、简化点或实验定位。

### 2.23 `Glandular_Morphometrics_for_Objective_Grading_of_Colorectal_Adenocarcinoma_Histology_Images_2017.pdf`

用途定位：

- 腺体形态学价值最直接的证据之一。
- 用于支撑“为什么做 gland segmentation 值得”，因为分割后的腺体边界可以进一步转化为肿瘤分级相关的可量化形态指标。

可直接提取的关键信息：

1. 论文开篇直接指出：`CRA` 的分级部分取决于 glandular morphology 和 gland formation，而传统病理分级存在主观性和一致性不足，因此需要 objective computer-aided measure。
   - 来源：`Glandular_Morphometrics_for_Objective_Grading_of_Colorectal_Adenocarcinoma_Histology_Images_2017.pdf`, p.2

2. 文中提出 `Best Alignment Metric (BAM)` 来量化腺体形状异常度，并证明 glandular shape 与肿瘤 grade 之间存在强相关。
   - 来源：`Glandular_Morphometrics_for_Objective_Grading_of_Colorectal_Adenocarcinoma_Histology_Images_2017.pdf`, p.2, p.4

3. 论文明确说：基于形状的分析前提是准确的 gland boundary detection，因此作者先用深度卷积网络完成 gland segmentation，再从分割结果中提取形态特征。
   - 来源：`Glandular_Morphometrics_for_Objective_Grading_of_Colorectal_Adenocarcinoma_Histology_Images_2017.pdf`, p.2, p.4

4. 关于病理标准，文章给出二级分级常用准则：若肿瘤中 `50%` 及以上区域仍呈腺体形成，则归为 low grade；低于 `50%` 则归为 high grade。
   - 来源：`Glandular_Morphometrics_for_Objective_Grading_of_Colorectal_Adenocarcinoma_Histology_Images_2017.pdf`, p.3

5. 数据协议部分说明：作者使用 `38` 张 `H&E` 染色 `WSI`，从中裁出 `139` 张 `20×` 图像，分为 `71` normal、`33` low grade、`35` high grade，并采用 `3-fold cross-validation`。
   - 来源：`Glandular_Morphometrics_for_Objective_Grading_of_Colorectal_Adenocarcinoma_Histology_Images_2017.pdf`, p.7

6. 分割训练中使用 `428×428` patch，做翻转、旋转和 elastic distortion 增强；推理阶段使用带重叠的 patch，并用 `alpha blending` 合并输出；预处理里采用 `Reinhard stain normalization`。
   - 来源：`Glandular_Morphometrics_for_Objective_Grading_of_Colorectal_Adenocarcinoma_Histology_Images_2017.pdf`, p.8, p.9

7. 后处理部分明确提到：对概率图阈值化后还需做形态学操作，以去除小目标、填洞并分离轻微黏连的目标，这说明 gland downstream morphology 对分割后处理质量非常敏感。
   - 来源：`Glandular_Morphometrics_for_Objective_Grading_of_Colorectal_Adenocarcinoma_Histology_Images_2017.pdf`, p.10

8. 最终作者报告：基于形状特征的两分类准确率达 `97%`，三分类准确率达 `91%`，说明 gland morphometrics 对 CRA grading 具有较强区分力。
   - 来源：`Glandular_Morphometrics_for_Objective_Grading_of_Colorectal_Adenocarcinoma_Histology_Images_2017.pdf`, p.2

对当前实验最有用的落地点：

- 这篇是你后面写“腺体分割的临床/病理意义”时非常强的一篇，因为它把分割结果直接连接到客观分级。
- 它还提示一个很重要的实验方向：即使主任务是 segmentation，也可以后续考虑把腺体形态特征作为下游验证价值。

### 2.24 `Semantic_Segmentation_of_Colon_Glands_with_Deep_Convolutional_Neural_Networks_and_Total_Variation_Segmentation_2015.pdf`

用途定位：

- 早期直接针对 colon gland segmentation 的任务内论文。
- 虽然年代较早，但它对 `GlaS` 早期数据口径、H&E 预处理、分离相邻腺体的实现思路仍有直接参考价值。

可直接提取的关键信息：

1. 论文明确面向 `GlaS@MICCAI2015` 的 colon gland segmentation 任务，强调 automated gland segmentation 对 morphology assessment 和 cancer grading 的支撑价值。
   - 来源：`Semantic_Segmentation_of_Colon_Glands_with_Deep_Convolutional_Neural_Networks_and_Total_Variation_Segmentation_2015.pdf`, p.1

2. 文中给出的早期 challenge 数据口径为：`Warwick-QU` 数据集含 `161` 张标注图像，染色为 `H&E`，扫描倍率为 `20×`；训练集 `85(37/48)`、测试集 A `60(33/27)`、测试集 B `16(12/4)`。
   - 来源：`Semantic_Segmentation_of_Colon_Glands_with_Deep_Convolutional_Neural_Networks_and_Total_Variation_Segmentation_2015.pdf`, p.3, p.4

3. 预处理部分采用 `color deconvolution` 分离 H&E 组织成分，只保留 deconvolved 图像的红色通道，并进一步做 `CLAHE` 以减小染色与光照差异。
   - 来源：`Semantic_Segmentation_of_Colon_Glands_with_Deep_Convolutional_Neural_Networks_and_Total_Variation_Segmentation_2015.pdf`, p.3, p.4

4. 方法不是单一 CNN，而是两个像素分类网络联合：`Object-Net` 负责 gland / background 预测，`Separator-Net` 专门学习 gland-separating structures，用来帮助分离紧邻腺体。
   - 来源：`Semantic_Segmentation_of_Colon_Glands_with_Deep_Convolutional_Neural_Networks_and_Total_Variation_Segmentation_2015.pdf`, p.3, p.6

5. `Object-Net` 不是简单二分类，而是把像素分成 `benign background / benign gland / malignant background / malignant gland` 四类，说明 benign 和 malignant 的纹理差异被显式纳入建模。
   - 来源：`Semantic_Segmentation_of_Colon_Glands_with_Deep_Convolutional_Neural_Networks_and_Total_Variation_Segmentation_2015.pdf`, p.4, p.5

6. 后处理不是普通阈值，而是把 CNN 输出作为输入，再用基于 `weighted total variation` 的 figure-ground segmentation 做全局最优的轮廓正则化。
   - 来源：`Semantic_Segmentation_of_Colon_Glands_with_Deep_Convolutional_Neural_Networks_and_Total_Variation_Segmentation_2015.pdf`, p.7

7. 实现细节里，作者把图像先缩小到一半分辨率，使用 `101×101` patch 做像素分类；同时为 `Separator-Net` 额外人工标注相邻腺体之间的分离结构，并通过旋转扩充样本。
   - 来源：`Semantic_Segmentation_of_Colon_Glands_with_Deep_Convolutional_Neural_Networks_and_Total_Variation_Segmentation_2015.pdf`, p.8

8. 训练时两个 CNN 都使用每类 `125,000` 个平衡采样 patch，说明在 gland segmentation 这种类别不均衡问题上，采样策略会直接影响训练稳定性。
   - 来源：`Semantic_Segmentation_of_Colon_Glands_with_Deep_Convolutional_Neural_Networks_and_Total_Variation_Segmentation_2015.pdf`, p.8

对当前实验最有用的落地点：

- 这篇很适合放在 early gland segmentation related work 里，说明“分离黏连腺体”很早就不是新问题。
- 它还给了你两个很实用的提醒：`H&E` 预处理确实重要；以及分离相邻腺体时，专门建模 separator/边界结构往往比只做单输出分割更稳。

### 2.25 `Efficient_Segmentation_Learning_Downsampling_Near_Semantic_Boundaries_2019.pdf`

用途定位：

- 泛语义分割方法学支撑，不是腺体任务直系论文。

1. 论文开篇明确指出：为了加速分割，很多方法会对输入做下采样，但这会带来两个直接代价，分别是 `small objects` 丢失和 `semantic boundaries` 精度下降。
   - 来源：`Efficient_Segmentation_Learning_Downsampling_Near_Semantic_Boundaries_2019.pdf`, p.1

2. 作者提出 `content-adaptive downsampling`，核心思想是让采样位置更多地分布在目标类的 semantic boundaries 附近，而不是做均匀下采样。
   - 来源：`Efficient_Segmentation_Learning_Downsampling_Near_Semantic_Boundaries_2019.pdf`, p.1, p.3

3. 论文理论和直觉都强调：在边界附近保留更多采样点，可以降低边界定位误差；同时这种非均匀采样还有助于提升小尺度目标的保留能力。
   - 来源：`Efficient_Segmentation_Learning_Downsampling_Near_Semantic_Boundaries_2019.pdf`, p.1, p.2

4. 整个框架分成三部分：`non-uniform downsampling`、任意现有 segmentation model、以及与之对应的 `non-uniform upsampling`，说明这是一种可插入现有分割系统的通用策略。
   - 来源：`Efficient_Segmentation_Learning_Downsampling_Near_Semantic_Boundaries_2019.pdf`, p.2, p.4

5. 文中反复强调，相比 uniform downsampling，该方法能在较小额外计算开销下持续改善 boundary quality，并对小目标更友好。
   - 来源：`Efficient_Segmentation_Learning_Downsampling_Near_Semantic_Boundaries_2019.pdf`, p.1, p.2

6. 论文还指出这种 boundary-aware 的非均匀采样具有“隐式尺度均衡”效果：会减少大区域在下采样图上的占比、增加小目标可见采样点，因此有助于避免小结构在均匀下采样时被直接漏掉。
   - 来源：`Efficient_Segmentation_Learning_Downsampling_Near_Semantic_Boundaries_2019.pdf`, p.2

7. 整个框架被设计成 `downsampling block + 任意 segmentation model + non-uniform upsampling` 三段式，因此它的价值更像一个可插拔前后处理策略，而不是必须重做主干网络。
   - 来源：`Efficient_Segmentation_Learning_Downsampling_Near_Semantic_Boundaries_2019.pdf`, p.2, p.3

对当前实验最有用的落地点：

- 这篇不需要像腺体主线论文那样详细复现，但很适合放进 `Boundary Head` 或边界质量讨论部分，说明分割网络一旦在下采样阶段损伤边界，后面很难完全补回来。
- 如果你后面讨论输入分辨率、边界细节、或为什么对象级边界指标重要，这篇可以当泛方法学支撑。

- `Gland_Segmentation_via_Dual_Encoders_and_Boundary-Enhanced_Attention_2024.pdf`

当前可直接形成的结论：

- 腺体任务里，边界监督不是可有可无，而是与分离黏连腺体直接相关
- 显式形状/边界分支是合理路线
- 面向 gland 的 attention 设计，很多时候本质上也是在提升边缘与相关区域的可分性
- 如果下采样阶段先伤害了边界和小目标，再强的解码器也很难完全恢复，因此边界附近的信息保留本身就是重要问题
- topology-aware 路线说明：除了边界，保持腺体骨架和拓扑结构也能帮助分离黏连目标
- 对象级结果不仅由网络决定，实例级后处理也会影响 `F1 / Object Dice / Object Hausdorff`
- distance-aware loss 更适合作为增强项，而不是一开始就默认必须保留的核心模块

### 3.4 大核与感受野

当前最关键的动机文献：

- `Large_Kernel_Matters_GCN_2017.pdf`

- `WTConv` 说明扩大感受野还有更高效路线，并且可能额外带来更强 shape bias
