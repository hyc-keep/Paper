# 服务器切换接力说明

## 1. 这份文档的用途

这份文档用于在下面这些场景里做上下文接力：

1. 从本地窗口切换到服务器窗口
2. 从一个 Trae 窗口切换到另一个 Trae 窗口
3. 重新打开项目后，需要快速恢复当前阶段上下文
4. 需要把“当前做到哪一步、下一步做什么、哪些文件最关键”快速告诉新的会话

这份文档不负责替代完整计划，也不替代阶段实现卡、研究定标记录或阶段状态页。

它只做一件事：

**把当前最重要、最容易丢失、最影响继续执行的信息压缩到一页里。**

这里按真正建议的执行顺序写，尽量写成 5-10 步以内的 checklist。

1. 在服务器上 `git clone https://github.com/hyc-keep/Paper.git` 拉取当前基线。
2. 打开新窗口后先读取 `Paper/服务器切换接力说明.md` 和 `Paper/02_UNet流程验证_执行检查清单.md`。
3. 在服务器上确认 `CUDA`、`PyTorch`、`AMP`、单卡显存和训练入口全部可用。
4. 先执行不入表 GPU smoke check，确认 forward、backward、optimizer.step 与 AMP 路径正常。
5. 通过后执行正式单 GPU + AMP 的 `A1_UNet_GlaS_v1_seed3407`。
